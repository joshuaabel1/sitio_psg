    listas:

    Una lista en Python es una estructura de datos que se utiliza para almacenar m√∫ltiples elementos en una √∫nica variable. Las listas son uno de los cuatro tipos de datos incorporados en Python que se utilizan para almacenar colecciones de datos, los otros tres son Tuple, Set y Dictionary.

    Las listas en Python tienen las siguientes caracter√≠sticas:

    1. Ordenadas: Las listas mantienen un orden espec√≠fico. Es decir, los elementos en una lista aparecen en un orden espec√≠fico. Por ejemplo, la lista ["manzana", "pl√°tano", "cereza"] siempre mantendr√° ese orden a menos que se modifique expl√≠citamente.

    2. Modificables (Mutables): A diferencia de las tuplas en Python, las listas pueden ser modificadas despu√©s de su creaci√≥n. Puedes agregar, eliminar y cambiar elementos de la lista.

    3. Permiten duplicados: Las listas permiten elementos duplicados. Por ejemplo, ["manzana", "pl√°tano", "cereza", "manzana"] es una lista v√°lida.

    Una lista se crea colocando todos los elementos (elementos) dentro de corchetes [], separados por comas. Por ejemplo:

    frutas = ["manzana", "pl√°tano", "cereza"]

    Puedes acceder a los elementos de la lista por su √≠ndice, comenzando por 0 para el primer elemento. Por ejemplo, frutas[0] devolver√° "manzana".

    Las listas tambi√©n soportan una serie de m√©todos para manipular su contenido, como append() para agregar elementos, remove() para eliminar elementos, sort() para ordenar la lista, entre otros.

    frutas = ["manzana", "banana", "cereza", "durazno", "frambuesa"]

    Acceder a un √≠ndice espec√≠fico: Para acceder a un elemento de la lista, usamos su √≠ndice de la siguiente manera:

    print(frutas[1])  # Salida: banana

    Acceder a un rango de √≠ndices: Para acceder a varios elementos de la lista a la vez, podemos usar el concepto de "slicing". Por ejemplo, si queremos acceder desde el √≠ndice 1 al √≠ndice 3:

    print(frutas[::-1])  # Salida: ['frambuesa', 'durazno', 'cereza', 'banana', 'manzana']

    Las listas en Python proporcionan varios m√©todos para manipular y trabajar con los datos.

    1. append(elemento): Agrega un elemento al final de la lista.
    lista = [1, 2, 3]
lista.append(4)
print(lista)  # Salida: [1, 2, 3, 4]
    1. extend(iterable): Extiende la lista agregando todos los elementos del iterable (otra lista, tupla, conjunto, etc.)
    lista1 = [1, 2, 3]
lista2 = [4, 5, 6]
lista1.extend(lista2)
print(lista1)  # Salida: [1, 2, 3, 4, 5, 6]
    1. insert(indice, elemento): Inserta un elemento en una posici√≥n espec√≠fica en la lista. El primer argumento es el √≠ndice del elemento antes del cual se insertar√°.
    lista = [1, 2, 4]
lista.insert(2, 3)
print(lista)  # Salida: [1, 2, 3, 4]
    1. remove(elemento): Elimina el primer elemento de la lista que coincida con el argumento.
    lista = [1, 2, 3, 2]
lista.remove(2)
print(lista)  # Salida: [1, 3, 2]
    1. pop(indice): Elimina el elemento en una posici√≥n espec√≠fica en la lista y lo devuelve. Si no se especifica un √≠ndice, pop() elimina y devuelve el √∫ltimo elemento de la lista.
    lista = [1, 2, 3, 4]
elemento_eliminado = lista.pop(1)
print(lista)  # Salida: [1, 3, 4]
print(elemento_eliminado)  # Salida: 2

    tuplas:

    Las tuplas en Python son estructuras de datos que se utilizan para almacenar varios elementos en una √∫nica variable.

    1. Inmutables: A diferencia de las listas, las tuplas son inmutables. Esto significa que, una vez que una tupla es creada, no puedes cambiar sus elementos ni su tama√±o.

    2. Ordenadas: Al igual que las listas, las tuplas mantienen un orden espec√≠fico. Es decir, los elementos en una tupla aparecen en un orden espec√≠fico.

    3. Permiten duplicados: Las tuplas permiten elementos duplicados.

    Una tupla se crea colocando todos los elementos (√≠tems) dentro de par√©ntesis (), separados por comas. Por ejemplo:

    frutas = ("manzana", "banano", "cereza")

    Puedes acceder a los elementos de la tupla usando su √≠ndice, que comienza en 0 para el primer elemento. Por ejemplo, frutas[0] devolver√≠a "manzana".

    A pesar de que las tuplas son inmutables, puedes tomar porciones de las tuplas existentes para crear nuevas tuplas. Tambi√©n puedes concatenar tuplas para formar nuevas. Aqu√≠ un ejemplo de cada caso:

    # Definimos una tupla
numeros = (1, 2, 3, 4, 5, 6)

# Tomar una porci√≥n de la tupla
sub_tupla = numeros[1:4]  # Esto es (2, 3, 4)

# Concatenar tuplas
tupla_concatenada = numeros + sub_tupla  # Esto es (1, 2, 3, 4, 5, 6, 2, 3, 4)

    Sets:

    Los sets en Python son una colecci√≥n de elementos √∫nicos, no ordenados y mutables. Son similares a las listas y las tuplas.

    1. No ordenados: Los sets no mantienen un orden espec√≠fico de los elementos. Cuando imprimes un set, los elementos pueden aparecer en cualquier orden, y el orden puede cambiar en diferentes ejecuciones del mismo c√≥digo.

    2. Mutables: Puedes a√±adir y eliminar elementos de un set despu√©s de su creaci√≥n.

    3. No permiten duplicados: Cada elemento en un set es √∫nico. Si tratas de a√±adir un elemento que ya existe en el set, Python simplemente lo ignorar√°.

    Un set se crea utilizando llaves {} o la funci√≥n set(). Por ejemplo:

    frutas = {"manzana", "banano", "cereza"}

    O si quieres crear un set vac√≠o:

    s = set()

    Nota: No puedes crear un set vac√≠o con {} porque Python lo interpretar√° como un diccionario.

    Puedes a√±adir elementos a un set utilizando el m√©todo add() y puedes eliminar elementos utilizando el m√©todo remove(). Por ejemplo:

    s = {1, 2, 3}
s.add(4)
print(s)  # Salida: {1, 2, 3, 4}

s.remove(1)
print(s)  # Salida: {2, 3, 4}

    Los sets tambi√©n soportan operaciones matem√°ticas como la uni√≥n, intersecci√≥n, diferencia y diferencia sim√©trica. Por ejemplo, si tienes dos sets:

    s1 = {1, 2, 3}
s2 = {2, 3, 4}

    Entonces puedes realizar operaciones como:

    print(s1.union(s2))  # Salida: {1, 2, 3, 4}
print(s1.intersection(s2))  # Salida: {2, 3}
print(s1.difference(s2))  # Salida: {1}
print(s1.symmetric_difference(s2))  # Salida: {1, 4}

    dicts:

    Un diccionario en Python es una estructura de datos no ordenada y mutable que almacena pares de valores: claves y valores. A cada clave le corresponde un valor, y las claves dentro de un diccionario deben ser √∫nicas. Aqu√≠ algunas caracter√≠sticas de los diccionarios:

    1. No ordenados: Al igual que los sets, los diccionarios no mantienen un orden espec√≠fico para sus elementos.

    2. Mutables: Puedes cambiar el valor de una clave existente y puedes a√±adir o eliminar pares de clave-valor despu√©s de que el diccionario se ha creado.

    3. Indexados por sus claves: A diferencia de las listas y las tuplas, que est√°n indexadas por un rango de n√∫meros, los diccionarios est√°n indexados por sus claves.

    Un diccionario se crea usando llaves {}, con los pares clave-valor separados por dos puntos :. Por ejemplo:

    mi_diccionario = {"nombre": "Juan", "edad": 30, "ciudad": "Madrid"}

    En este ejemplo, "nombre", "edad", y "ciudad" son las claves, y "Juan", 30, y "Madrid" son sus respectivos valores.

    Para acceder a un valor en un diccionario, utilizas la clave correspondiente. Por ejemplo:

    print(mi_diccionario["nombre"])  # Salida: Juan

    Para a√±adir un nuevo par clave-valor al diccionario, simplemente asignas un valor a una nueva clave:

    mi_diccionario["ocupacion"] = "Ingeniero"
print(mi_diccionario)  # Salida: {'nombre': 'Juan', 'edad': 30, 'ciudad': 'Madrid', 'ocupacion': 'Ingeniero'}

    Y para eliminar un par clave-valor del diccionario, puedes usar la declaraci√≥n del:

    del mi_diccionario["edad"]
print(mi_diccionario)  # Salida: {'nombre': 'Juan', 'ciudad': 'Madrid', 'ocupacion': 'Ingeniero'}

    Los diccionarios son muy √∫tiles cuando necesitas asociar valores con claves espec√≠ficas, como cuando almacenas propiedades de un objeto en particular.

    excepciones:

    En Python, las excepciones son errores que ocurren durante la ejecuci√≥n del programa. Cuando ocurre una excepci√≥n, la ejecuci√≥n del programa se interrumpe. Python proporciona varias excepciones incorporadas, como IndexError, TypeError, ValueError, etc., y tambi√©n permite crear excepciones personalizadas.

    Por ejemplo, IndexError es una excepci√≥n incorporada que se lanza cuando intentas acceder a un √≠ndice que est√° fuera del rango v√°lido en una secuencia, como una lista o una tupla.

    ejemplo de c√≥mo podr√≠a ocurrir un IndexError:

    lista = [1, 2, 3]
print(lista[3])  # Lanza un IndexError porque los √≠ndices v√°lidos son 0, 1, 2

    Para manejar excepciones en Python, puedes usar una estructura try/except. El c√≥digo que podr√≠a lanzar una excepci√≥n se coloca dentro del bloque try, y el c√≥digo que maneja la excepci√≥n se coloca dentro del bloque except.

    un ejemplo de c√≥mo manejar un IndexError:

    lista = [1, 2, 3]

    print(lista[3])  # Intenta imprimir un elemento fuera del rango
except IndexError:
    print("√çndice fuera de rango")  # Maneja el error e imprime un mensaje de error

    En este ejemplo, cuando se produce un IndexError, el programa no se detiene. En lugar de eso, imprime "√çndice fuera de rango" y contin√∫a con la ejecuci√≥n del programa.

    Claro, aqu√≠ tienes un ejemplo en el que se solicita un √≠ndice al usuario a trav√©s de un input(), y se manejan las excepciones IndexError y ValueError:

    indice = int(input("Ingresa un √≠ndice para la lista: "))

    print("Error: √çndice fuera de rango.")
except ValueError:
    print("Error: No ingresaste un n√∫mero entero v√°lido.")

    En este c√≥digo, el programa solicita un √≠ndice al usuario. Intenta convertir el input del usuario en un entero y usarlo como √≠ndice para la lista.

    Si el usuario ingresa un n√∫mero que est√° fuera del rango de los √≠ndices de la lista, se lanza un IndexError, que se maneja imprimiendo "Error: √çndice fuera de rango."

    Si el usuario ingresa algo que no puede ser convertido en un n√∫mero entero (como un string de texto), int(input()) lanza un ValueError. Este error se maneja imprimiendo "Error: No ingresaste un n√∫mero entero v√°lido."

    De esta forma, el programa es capaz de manejar estos errores de forma elegante y proporcionar un feedback √∫til al usuario.
